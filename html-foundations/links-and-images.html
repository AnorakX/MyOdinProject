Links

This is an "a", or anchor element. It doesn't have any attributes, so it doesn't do anything.

By using the "href" attribute with an "a", I can link this to Google. This is called an absolute link, a link to another website on the Internet that contains the protocol and domain of the destination.

Using a file path relative to the page you are creating the link on, you can create a relative link. For example, this link to our lesson on lists, "lists.html". The full file path, "html-foundations/lists.html", however, will not work, as it is relative to the current directory. However, "../html-foundations/lists.html" brings us back to the parent directory, with "../" representing the parent directory. On the other hand, "./" represents the current directory, e.g. "./lists.html", though it's usually not necessary.

Images

Images are created with the "img" element. It's self-closing (no ending tag needed) and uses the "src" or source attribute for its image source.

Much like links, src accepts two types of links: absolute or relative. The src of the image below is an absolute link. (https://www.theodinproject.com/mstile-310x310.png)

The src of the next image is a relative link, which uses the same file path rules discussed above.

The img element also has the "alt" attribute, which is used to describe an image in place of the image if it can't be loaded. It's also used by screen readers for visually impaired users. Your img element should always have a src and alt attribute.

Main Image Formats

JPEG/JPG - raster-based

Pros:

  • Universal browser support
  • Small file and email attachment size, fast loading
  • Supports displaying millions of colors
  • Best used for sharp, good quality photographs

Cons:

  • Does not support transparent backgrounds
  • Lossy image compression leads to low quality or poor text readability
  • Not to be used for computer-generated graphics
  • Must be saved with the correct size and resolution to avoid pixelation and image stretching

PNG - pixel-based

Pros:

  • Universal browser support
  • Supports transparency
  • Best used for graphical elements - logos, icons, simple illustrations
  • Lossless compression
  • Small file size with limited colors and rapid transitions between colors that remain sharp

Cons:

  • Large file size with millions of colors
  • Not ideal for printing - optimized for the screen

WebP

Pros:

  • Smaller file size for the same image quality (35% in comparison to JPEG, 50% to PNG)
  • Uses both lossy and lossless compression
  • Supports transparency

Cons

  • Not supported by all browsers, specifically Safari and Internet Explorer
  • Not supported by all CMS platforms

SVG - vector-based

Pros:

  • Universal browser support
  • Small file size
  • Loss-free scaling for basic shapes and text
  • Supported by illustration software
  • Supports transparency
  • Best used for logos, icons, and simple illustrations because of the sharp quality graphics

Cons:

  • Not an ideal format for images or complex drawings
  • Not supported by all CMS platforms, but there are workarounds
  • Not supported by some default image editors

GIF - animated image

Pros:

  • Animation support: used for small animations and low-resolution video clips
  • Universal browser support
  • Small file size

Cons:

  • Loss of quality of photographs
  • Limited colors due to 8-bit (PNG and JPG supports 24-bit)
  • Copyright issues
